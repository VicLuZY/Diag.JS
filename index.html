Under Development

Diag.JS, its DSL, renderer behavior, device libraries, and documentation are still evolving. Expect syntax, examples, supported equipment, and output conventions to change as the platform matures.

Diag.JS / Text-First Diagram Platform

A shared DSL, renderer stack, and documentation system for maintainable infrastructure diagrams.

Write system intent once. Render diagrams teams can review, ship, and extend.

Diag.JS is for engineers, integrators, and library authors who want diagrams to behave like maintainable source instead of one-off drawings. Today it turns a shared DSL into exportable schematics across power, HVAC, network, fire alarm, and lighting control. As the platform develops, the same foundation is meant to support broader system coverage, stricter validation, richer libraries, and deeper reference tooling.

Documentation

Treat the docs as the contract for the language, libraries, and future growth of the platform.

The documentation site is organized for both daily use and extension work: how to write valid DSL, how each renderer interprets it, how equipment types and connection classes are defined, and what standards new systems should follow as the platform expands. It is built with VitePress and published alongside the app at /docs/.

How Teams Use It

Author, review, and scale from the same source

  • Draft a model in text and keep it diffable in version control.
  • Review the rendered SVG beside the source instead of maintaining separate drawing files.
  • Reuse one language across multiple systems while keeping renderer-specific semantics where they matter.
  • Expand the platform through documented equipment libraries and generated reference pages.
