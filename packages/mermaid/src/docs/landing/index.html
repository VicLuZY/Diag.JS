DiagJS

Single-line diagrams (SLD), flowcharts, and automatic layout. A focused fork of Mermaid for electrical device and cable semantics.

View documentation →

About

DiagJS adds the sld diagram type with four-character type definitions (for example ATSW, GENR, UTIL, CDPN) and cable definitions, while keeping optional compatibility with flowchart and graph syntax. Layout is handled by dagre (default) or ELK via config.

This project preserves Mermaid attribution and remains under the MIT License. See the docs and the repository for more.

Example: Single-line diagram (SLD)

Building power from utility and generator through an ATS, switchgear, feeder, and panel to loads.

Rule: all SLD type definitions must be exactly 4 uppercase characters.

Open the docs to see this diagram rendered on the home page and in the live editor. 

sld LR
%% Rule: all type definitions use exactly 4 uppercase characters.
UTIL1[Utility]{v="480V"}
GENR1[Generator]{v="480V"}
ATSW1[Main ATS]{profile="basic", out_breaker=true}
SWGR1[Switchgear]{v="480V"}
UTIL1 --> ATSW1:norm
GENR1 --> ATSW1:emer
ATSW1:load ==> SWGR1
SWGR1 -->|Feeder F-12| COND1{type="EMTC", len="12m"} --> CABL1{cond="4C", size="3/0"} --> PANL1[Panel L1]{v="480V"}
PANL1 --> BRKR_PANL1_1 --> LGHT1[Lighting]
PANL1 --> BRKR_PANL1_2 --> RCPT1[Receptacles]
